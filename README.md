# Block-Trade-Quant-Strategy
# A股大宗交易溢价因子策略 v8g

溢价大宗交易信号 + LLT动态仓位管理回测框架。

## 核心策略

**选股条件：** 大宗交易溢价率 >= 5% + 市值 < 100亿

**仓位管理：** 基于 LLT（低延迟趋势线）的市场状态动态分配单只仓位

| LLT状态 | 单只仓位 | 含义 |
|:-------:|:-------:|:----|
| **BULL** | 5% | 上升趋势，轻仓防追高 |
| **NEUTRAL** | 25% | 震荡市，中性仓位 |
| **BEAR** | 35% | 下跌趋势，高仓位吃折价安全垫 |

**BULL 额外过滤：** 价格高于 MA5/MA10 时跳过不买

**持有期：** 30天，开盘价买入 / 开盘价卖出

## 运行方法

### 1. 数据准备

策略需要以下数据文件（因体积较大，需自行准备）：

- `data/wind_data_full/block_trades.csv` — 大宗交易数据
- `data/wind_data_full/daily_prices.csv` — 日K线数据（含前复权因子）
- `data/wind_data_full/derivative_indicators.csv` — 衍生指标（PE/PB/市值/换手率）

数据来源：Wind 金融终端或 Wind MySQL 数据库。

### 2. 运行回测

```bash
# 安装依赖
pip install -r requirements.txt

# 修改 config.py 中的 DATA_DIR 路径指向你的数据目录
# 运行回测
python scripts/19_backtest_v8g.py
```

### 3. 实时信号

```bash
# 需要配置 Wind MySQL 只读连接
python scripts/22_v8g_live_signals.py
```

## 关键发现

1. **BEAR 状态贡献最大利润**：大宗折价提供了足够安全垫，BEAR 时不应减仓
2. **BULL 状态下容易追高**：BULL + 价格在 MA 上方时买入，亏损概率高
3. **换手率因子**：与大宗交易后收益显著负相关（Rank IC -0.08~-0.12）
4. **亏损黑名单弊大于利**：会错过"先亏后赚"的大盈利交易（如东方园林 +24万）
